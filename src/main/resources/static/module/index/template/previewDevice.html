全屏预览: 设备操作区域全屏显示。
开始预览: 发送指令,传输手机端图像显示到界面,可通过右侧常用参数修改刷新率、图片质量等。
停止预览: 发送指令,停止传输手机端图像。
屏幕方向: 所选设备的屏幕方向,手机端旋转屏幕时,会自动监听变化。(如果监听失效,可手动修改)
APP刷新: 预览图像的频率,数值越低刷新频率越高,越占用带宽。(局域网可以设置为30,部署公网带宽低的不建议设置过低)
预览质量: 预览图像的压缩质量,数值越低图像越模糊,响应速度越快。(最大值为100,默认10,为提高响应速度可设置为6)
缩放倍数: 预览图像的缩放备注,数值越低图像越模糊,响应速度越快。(默认为1,不缩放,为提高响应速度可设置1以下)
预览阈值: 开启阈值化后,调整阈值显示。(主要用于开发图色脚本时,实时测试阈值效果)
文本框序号: 传输文本到设备时,指定输入框序号,为空则发送给所有输入框。(仅支持常规输入框控件)
传输类型: 传输文本到设备,追加文本或者覆盖文本内容。
灵敏模式: 常规预览模式,web端绘制完成指令后发送给APP执行,有一定的延迟。灵敏模式,web端实时绘制指令并发送到APP执行,如按下、移动、松开模拟滑动操作。(需要adb权限或者root权限支持)
开启灰度化: 开启灰度化后,预览设备显示为灰度化效果。(主要用于开发图色脚本时,实时测试灰度化效果)
开启阈值化: 开启阈值化后,调整阈值显示。(主要用于开发图色脚本时,实时测试阈值效果)
运行中脚本: 查看APP中正在运行的AutoJs脚本,支持停止脚本。
定时任务: 查看APP中已开启的定时任务,支持停止定时任务。
通知监听: 根据规则配置，监听设备的指定包名包含指定内容的通知,符合条件的可以支持推送消息到Web、APP控制面板消息、邮件转发、自定义代码执行等。下方表格还可以查询服务端启动后，监听成功的历史通知。
